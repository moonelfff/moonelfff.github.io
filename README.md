## Изменение содержимого

### Структура файлов

Большая часть контента находится в **[src/assets/data/content.json](src/assets/data/content.json)**.

Длинные тексты вынесены в папку `public` и находятся в файлах с расширением `.md`. Пример: **[/public/profile.md](/public/profile.md)**. Такие тексты можно дополнительно форматировать.

### content.json (общая информация, прайс)

У каждого значения есть свой ключ, по которому оно подставляется, например `"fullName": "Екатерина Жук"`. Любое из значений можно менять.

Прайс состоит Чтобы добавить новый продукт в прайс, необходимо в `"pricelist": [ ... ]` добавить новую структуру аналогично существующим. Порядок данных в файле - это порядок их отображения на сайте. Структура должна быть такой:

```json
{
  "descriptionPath": "/pricelist/имя_файла_с_описанием.md", // обязательно проверить имя файла
  "packages": [ // список пакетов (минимум один элемент)
    {
      "name": "1 встреча", // имя пакета
      "price": {
        "regular": { "amount": 3333, "currency": "₽" } // обычная цена и валюта
      }
    },
    {
      "name": "4 встречи", // имя пакета
      "price": {
        "regular": { "amount": 9999, "currency": "₽" }, // обычная цена
        "discounted": { "amount": 8888, "currency": "₽" } // цена со скидкой (опционально)
      }
    }
  ],
  "title": "Разовая консультация"
}
```

### Markdown (длинный текст с форматированием)

> Шпаргалка по Markdown [тут](https://gist.github.com/fomvasss/8dd8cd7f88c67a4e3727f9d39224a84c)

В данный момент на сайте поддерживаются заголовки, обычный текст и маркированные списки. По запросу можем добавить поддержку других элементов.

Все файлы с текстом хранятся в папке `public`, цены в папке `public/pricelist`. Ссылка на каждый из файлов должна присутствовать в `content.json`, иначе текст не будет отображаться.
